Threadbox — Agent DSL

Imperative builder → IR → validation → transpile → load → run. Deterministic, auditable, and budgeted orchestration with a typed blackboard and bounded epochs.

GitHub (main) Technical Report (PDF)

Overview

This project provides a TypeScript DSL that describes topology first (seq, inParallel, andThen, join) and emits an intermediate representation (IR) that we can validate against a registered tool/agent ABI before transpiling to executable code. Agents are just constrained call sites with schema'd IO and budgets; optional blackboard coordination is typed and append-only. Replanning is expressed as bounded epochs with explicit progress measures—no open-ended loops.

For background and related engineering patterns see the orchestration and state-machine write-ups (influences and contrasts).
